How to access DLAB user environment on different operation systems

    For Windows

    How to setup tunnel to EDGE instance via SSH

  • Please download PuTTY program by using this link.
  • Open the PuTTY program.
  • In the "HostName" field enter user and IP address of your Edge Node. Please use "ubuntu" user for Ubuntu-base environment and "ec2-user" for RedHat.
  • Open Connection → SSH → Auth tab and select your private key
  • Open Connection → SSH → Tunnels tab. In "Source port" field enter "3128". In "Destination" select "dynamic". Click "Add" button
  • Open Session tab. In "Saved Session" field enter name of the connection. Press "Save" and "Open" buttons
  • Tunnel to the Edge has been established. Do not close this terminal till you finished working with notebooks

    • How to access Notebook Server over SSH

  • Tunnel to the Edge should be established. Run a second PuTTY session.
  • In the "HostName" field enter user and IP address of your Notebook instance. lease use "ubuntu" user for Ubuntu-base environment and "ec2-user" for RedHat.
  • Open Connection → SSH → Auth tab and select your private key
  • Open Connection → Proxy tab. Select "SOCKS4" Proxy type. Enter "127.0.0.1" and "3128" in "Proxy hostname" and "Port" fields respectively
  • Open Session tab. In "Saved Session" field enter name of the connection. Press "Save" and "Open" buttons
  • Now you are connected to notebook server via SSH.

    • How to access Notebook Server over HTTP

  • Tunnel to the Edge should be established.
  • Go to Control Pannel → Lan Settings → Internet properties → Connection tab
  • Select "use proxy" and press "advanced" button.
  • In the "Socks" string enter "127.0.0.1" and "3128" in "Proxy address" and "Port" fields respectively. Press OK
  • Now you are able to access Notebook server URL via HTTP using any browser.

    • For MacOS / Linux

    How to setup tunnel to EDGE instance via SSH

  • Open terminal.
  • Enter following command":
    • ssh -D 3128 USER@EDGE_IP -i PATH_TO_PRIVATE_KEY
  • Please change USER to "ubuntu" for Ubuntu-base environment and to "ec2-user" for RedHat. Also change "EDGE_IP" and "PATH_TO_KEY" to your value.
  • Tunnel to the Edge has been established. Do not close this terminal till you finished working with notebooks

    • How to access Notebook Server over SSH

  • Tunnel to the Edge should be established. Open new terminal.
  • Enter following command":
    • ssh -o ProxyCommand='nc -x localhost:3128 %h %p' USER@NOTEBOOK_IP -i PATH_TO_PRIVATE_KEY
  • Please change USER to "ubuntu" for Ubuntu-base environment and to "ec2-user" for RedHat. Also change "NOTEBOOK_IP" and "PATH_TO_PRIVATE_KEY" to your value
  • Now you are connected to notebook server via SSH.
